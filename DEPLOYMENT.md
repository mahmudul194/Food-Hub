# 🚀 FoodHub Deployment Guide

This guide will help you deploy your full-stack FoodHub platform to a free hosting service.

## Recommended: [Render.com](https://render.com) (Free Tier)

Render is an excellent choice for hosting the Node.js backend because it serves your frontend and backend as one unit.

### 1. Database Setup (MySQL)
Cloud providers often provide a single **Connection String**. You can get a free MySQL database from:
1.  **TiDB Cloud**: Offers a generous free tier for MySQL-compatible databases.
2.  **Aiven**: Offers a free MySQL plan.

Keep your **Connection String** ready (e.g., `mysql://user:pass@host:port/dbname`).

### 2. Prepare the Code
1.  Initialize a Git repository in your project folder:
   ```bash
   git init
   git add .
   git commit -m "Prepare for deployment"
   ```
2.  Push your code to **GitHub**.

### 3. Deploy to Render
1.  Log in to [Render](https://dashboard.render.com).
2.  Click **New +** > **Web Service**.
3.  Connect your GitHub repository.
4.  **Configuration**:
    - **Name**: `foodhub-app`
    - **Environment**: `Node`
    - **Build Command**: `cd backend && npm install`
    - **Start Command**: `cd backend && npm start` (I have already configured this for you!)
5.  **Environment Variables**:
    Under the **Environment** tab, add:
    | Key | Value |
    | :--- | :--- |
    | `DATABASE_URL` | *Your MySQL connection string* |
    | `NODE_ENV` | `production` |

### 4. How it Works
The backend server (`server.js`) is already optimized to serve your frontend files. This means once you deploy the backend, your entire website will be live at the URL Render provides.
